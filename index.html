Metadata: elements that contain information about the page that helps software use and renders the page. This is contained in the head section of an HTML document.

>
Element Description
<HTML> The top-level, 'root element' of an HTML document. HTML 5 is the current standard for HTML.
<head>Contains metadata about the document, including the title and stylesheets.
<link> Specifies the relationship between the HTML document and an external resource, such as a CSS stylesheet.
<style> Contains style information using CSS.
<title> Defines the document's title as shown in the page's tab/brower's title bar.

Content sectioning: elements used to organise the document logically and create an outline for the page's content.

Element Description
<body> Represents the content of an HTML document
<header> Contains introductory content, such as navigational aids, a logo, a search form or an author's name.
<nav> Section in the header that contains navigation to parts of the current document or to other documents. Examples include menus, indexes and tables of contents.
<h1>,<h2>,<h3>,<h4>,<h5>,<h6> The 6 levels of section headings, with < h1 > being the highest and < h6 > the lowest.
<main> Represents the dominant content of the body of the document, consisting of content directly related to the central topic.
<section> Represents a generic, standalone part of a document that lacks a more specific semantic element to represent it, but contains thematically similar content. Sections should always have a heading either above or below the element.
<article> An item of content, such as a blog post, magazine, or newspaper article, which is intended to be independantly distributed from the other content on the page.
<aside> Represents a section whose content is only indirectly related to the central topic of the document. Often presented as sidebars or call-out boxes.
<footer> Represents a footer for its nearest ancestor sectioning content/element, and typically contains information about the author of the above section, copyright data or related links.

Text Content: elements placed between the opening/closing <body> tags to organise & identify the purpose sections of content to improve readability and accessability.

Element Description
<div> A generic container for content that has no effect on the layout until styled with CSS.
<p> Represents a paragraph, which in HTML refers to any structural grouping of related content, even images or form fields.
<ul> An unordered list of items, automatically rendered as bullet points.
<ol> An ordered list of items, automatically rendered as a numbered list.
<li> Represents a list item, and must be contined in a <ul> or <ol> parent element.
<figure> Represents self-contained content, with an optional caption specifies using the <figcaption> element - these two elements are referenced as a single unit.
<figcaption> Represents a caption describing it's parent <figure> element.
<blockquote> Represents enclosed text that is an extended quotation, and may be followed by a source URL given using cite attribute or element.

Inline Text Semantics

Element Description
<a> Together with it's href attribute creates a hyperlink to web pages, files, locations within the current document, etc.
<br& Produces a line break in the text.
<em> Renders emphasised/italicised text.
<s> Renders text with a strikethrough
<small> Represents side comments and small print, and renders text within one font size smaller (small to x-small).
<span> An inline container that doesn't render, but can be used to target elements for styling purposes when no other semantic element is appropriate. It is the inline-level equivalent of the div element.
<strong> Indicates the importance of contents, and renders in bold.

Image and Multimedia: elements for multimedia resources including images, audio and video.

Element Description
<img> Embeds an image into the document.
<video> Embeds a media player which supports video playback into the document.
<audio> Embeds sound content in documents, and may contain one or more audio sources using the srcattribute
<area> Defins an area inside an image map that has a defined, clickable area, allowing geometric areas to be hyperlinked.

Table Content: elements used to create tabular data.

Element Description
<caption> Specific the title of a table.
<table> Represents all tabular data.
<thead> Defines a set of rows defining the head of the columns of the table.
<tbody> Contains a set of table rows that comprise the body of the table.
<th> Defines a cell as a header of a group of table cells.
<colgroup> Defines a group of columns within a table.
<col> Defines a column within a table, usually found within a <colgroup> element.
<tr> Defines a row of cells in a table, which can be categorised using a mix of <td> and <th> elements.
<td> Defines a cell of a tabe that contains data.
<foot> Defines a set of rows summarising the columns of the table.

Forms: elements used to create forms that a user can fill out and submit to the website/application.

Element Description
<form> Represents a document section containing interactive controls for submitting information.
<label> Represents a caption for an item in a user interface.
<input> Used to create interactive controls for forms to accept data from the user.
<output> Container element into which the site can inject the results/outcome of a user action.
<button> An interactive element activated by the user that performs an action, such as submitting a form.
<option> Used to define an item contained in a <datalist>, <select> or <optgroup> element, for example; menu items.
<datalist> Contains a set of <option> elements that represent the available options to choose from.
<select> Represents a control that provides a menu of options.
<optgroup> Creates a grouping of options within a <select> element.
<fieldset> Used to group several controls and labels.
<legend> Represents a caption for the content of its parent <fieldset>.
<meter> Represents a scalar value or fractional value within a known range.
<progress> Renders an indicator showing the completion progress of a task.
<textarea> Represents a multi-line plain-text editing control, used to allow users to enter a sizeable amount of text, such as a review.
